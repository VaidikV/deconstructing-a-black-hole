Deconstructing
a Black Hole

A deep dive into the fragment shader behind this animation.

Scroll to begin

01

What You're Looking At

The animation behind this page isn't a video. It isn't a GIF. It isn't pre-rendered. It's a fragment shader — a small program running on your GPU right now, computing the color of every pixel on screen, sixty times per second.

A fragment shader receives a pixel coordinate and returns a color. That's it. No loops over objects, no scene graph, no physics engine. Just math that maps position to color. The complexity you see — the grain, the swirl, the shape of the black hole — all of it emerges from a few hundred lines of GLSL running in parallel across millions of pixels.

This article breaks that shader apart, layer by layer. By the end, you'll understand every line. No prior shader experience required.

02

The Setup

The renderer is Three.js with WebGL. But the 3D engine does almost nothing here — there are no models, no lighting, no materials in the traditional sense. We use Three.js purely as a bridge to get our shader onto the GPU.

The scene has exactly two objects:

  1. A background plane — a flat rectangle that fills the screen with a solid dark color. Think of it as the "wall" behind the effect.
  2. A grain plane — another flat rectangle layered in front, where the actual shader runs. This plane is transparent — it outputs an alpha channel, so wherever the effect doesn't draw, the dark background shows through.

The camera is orthographic — no perspective distortion. The grain plane is always the same size on screen regardless of distance. This matters because the shader works in UV coordinates (0 to 1 across the surface), and we need that mapping to be stable.

JavaScript — Scene setup

const scene  = new THREE.Scene();
const camera = new THREE.OrthographicCamera(
  -1, 1,           // left, right
  1/aspect, -1/aspect,  // top, bottom (corrected for aspect ratio)
  0.1, 1000         // near, far
);

// Background: solid color, z = 0
const bgMesh = new THREE.Mesh(
  new THREE.PlaneGeometry(4, 4),
  solidColorMaterial
);

// Grain effect: transparent shader, z = 1 (in front)
const grainMesh = new THREE.Mesh(
  new THREE.PlaneGeometry(3, 3),
  new THREE.RawShaderMaterial({
    uniforms, vertexShader, fragmentShader,
    transparent: true
  })
);

The vertex shader is a pass-through — it just forwards the UV coordinates to the fragment shader. All the interesting work happens per-pixel.

03

Noise

The foundation of the entire effect is simplex noise — a mathematical function that takes a 2D coordinate and returns a smooth, random-looking value between -1 and 1. Unlike true randomness (which would look like TV static), simplex noise is coherent: nearby points return similar values, creating organic gradients and blobs.

We use the Ashima Arts implementation — a well-known open-source simplex noise that runs efficiently on GPUs. The raw function snoise(vec2) returns values in [-1, 1]. We remap it to [0, 1] with a wrapper:

GLSL — Noise primitives

float snoise01(vec2 v) {
  return (1.0 + snoise(v)) * 0.5;
}

float noise2d(vec2 st) {
  return snoise01(vec2(
    st.x + time * 0.02,       // drift right slowly
    st.y - time * 0.04 + seed  // drift down, offset by random seed
  ));
}

noise2d adds time to the input coordinates, which makes the noise drift — this is what creates the slow, evolving motion. The seed uniform is a random number set once on page load, so every visitor sees a unique pattern.

Domain Warping

Raw noise looks like blurry blobs. To get the complex, swirling, almost-organic patterns, we use a technique called domain warping — we use noise to distort the input of other noise.

GLSL — Domain warping (the pattern function)

float pattern(vec2 p) {
  // Layer 1: two noise samples offset in space
  vec2 q = vec2(
    noise2d(p + vec2(0.0, 0.0)),
    noise2d(p + vec2(5.2, 1.3))
  );

  // Layer 2: use q to warp the coordinates of new noise
  vec2 r = vec2(
    noise2d(p + 4.0*q + vec2(1.7, 9.2)),
    noise2d(p + 4.0*q + vec2(8.3, 2.8))
  );

  // Layer 3: final noise, warped by r
  return noise2d(p + 1.0*r);
}

This is noise of noise of noise. Each layer feeds into the next, amplifying the distortion. The magic numbers (5.2, 1.3, 1.7, 9.2...) are arbitrary offsets that prevent the layers from correlating — they just need to be "far apart" in noise space. The result is something that looks like slow turbulence: ink in water, smoke in air.

04

The Grain Texture

If domain warping creates the flow, the grain texture creates the texture — the gritty, particulate quality that makes the light look physical rather than digital.

grain.webp is a 1024×1024 image, but it doesn't contain a picture. It encodes two values per pixel:

  • Red channel — magnitude. How far this particle should scatter from its position.
  • Green channel — angle. The direction of the scatter, from 0 to 2π.

Together, they define a polar displacement vector for each pixel. The shader samples this texture in pixel space (not UV space), so the grain pattern is density-independent — it looks the same regardless of the mesh size.

GLSL — Grain sampling and polar displacement

// Sample grain at pixel coords, tiled to 1024px
vec3 grainColor = texture2D(grainTex,
  mod(p * param1 * 5.0, 1024.0) / 1024.0).rgb;

// Convert R (magnitude) and G (angle) to cartesian offset
float gr = pow(grainColor.r, 1.5) + 0.5 * (1.0 - blurAlpha);
float gg = grainColor.g;
float ax = param2 * gr * cos(gg * 2.0 * PI);
float ay = param2 * gr * sin(gg * 2.0 * PI);

The pow(grainColor.r, 1.5) compresses the magnitude distribution — most particles scatter a little, a few scatter a lot. The offset ax, ay is then added to the noise coordinates before calling pattern(). This means each pixel evaluates the noise at a slightly different location, breaking up the smooth noise field into a grainy, particulate texture.

The term 0.5 * (1.0 - blurAlpha) increases scatter outside the shape, which softens the boundary. Inside the shape where blurAlpha = 1, scatter is tighter and the grain is denser.

05

The Shape

The black hole isn't an image. It's not loaded from a file. It's built entirely from math inside the fragment shader — a procedural shape mask called blurAlpha.

The mask has two components:

  1. The ring — a donut shape with an inner radius (the "hole") and an outer radius that varies with elevation angle, creating the illusion of a 3D torus seen at an angle.
  2. The accretion disk — a thin horizontal band that tapers as it extends outward, like matter spiraling into the black hole.

Before computing the shape, the coordinate system is rotated about 29 degrees and scaled, which tilts the black hole across the viewport.

GLSL — Black hole shape mask

// Translate to center, rotate ~29°, scale
vec2 d = uv - vec2(0.63, 0.50);
float ca = cos(0.50), sa = sin(0.50);
d = vec2(ca * d.x + sa * d.y, -sa * d.x + ca * d.y);
d *= 0.79;

float dist  = length(d);
float holeR = 0.10;  // inner void radius

// Ring: outer radius varies with "elevation" for 3D illusion
float sinEl  = d.y / (dist + 0.001);
float outerR = mix(0.16, 0.135, (sinEl + 1.0) * 0.5);
float equator = 1.0 - abs(sinEl);
outerR += equator * equator * 0.015;  // slight bulge at equator

float ring = smoothstep(holeR, holeR + 0.02, dist)
           * (1.0 - smoothstep(outerR, outerR + 0.06, dist));

// Disk: horizontal band that tapers outward
float taper = mix(0.07, 0.015, smoothstep(0.12, 0.35, abs(d.x)));
float hBand = (1.0 - smoothstep(0.0, taper, abs(d.y)))
            * (1.0 - smoothstep(0.22, 0.40, abs(d.x)));

float blurAlpha = max(ring, hBand);

smoothstep is the workhorse here. It creates smooth transitions — the ring doesn't have a hard edge, it fades from 0 to 1 over a small range. The inner smoothstep carves out the hole; the outer one fades the ring away. Multiplying them creates the donut.

The sinEl trick approximates viewing angle: at the "top" and "bottom" of the ring, the outer radius shrinks; at the "sides" (equator), it's widest. This single line of math creates the illusion that you're looking at a 3D torus from a slight angle.

The accretion disk (hBand) is simpler — a horizontal stripe that's thick near the center and tapers to nothing at the edges. max(ring, hBand) combines them: wherever either shape is active, particles are visible.

06

The Threshold

At this point we have a noise value n between 0 and 1 for every pixel. If we used it directly as brightness, we'd see a smooth, cloudy wash of gray. The effect would be soft and lifeless.

The threshold step is what gives the grain its character — sparse bright particles against deep darkness.

GLSL — The contrast curve

float n = pattern(vec2(nx, ny));  // value in [0, 1]

n = pow(n * 1.05, 6.0);   // crush everything below the peak
n = smoothstep(0.0, 1.0, n);  // clamp and smooth

pow(x, 6.0) is a steep contrast curve. A noise value of 0.5 becomes 0.56 = 0.016 — practically invisible. A value of 0.8 becomes 0.86 = 0.26 — faintly visible. Only values very close to 1.0 survive with real brightness. The result: most of the noise field goes dark, and only the brightest peaks remain as visible particles.

The 1.05 multiplier is a subtle bias — it pushes all values slightly up before the power function, which controls the overall density of visible grain. Increasing it shows more particles; decreasing it makes the effect sparser.

The final smoothstep clamps the result to [0, 1] and adds a slight S-curve, ensuring clean output values for the color mix.

This single line — pow(n * 1.05, 6.0) — is arguably the most important line in the shader. It transforms amorphous noise into something that looks like physical particles of light.

07

Color and Compositing

The final step blends everything into a visible result.

GLSL — Final output

vec3 front = vec3(0.84, 0.86, 0.80);  // warm off-white
vec3 result = mix(back, front, n);    // blend by intensity

gl_FragColor = vec4(result, blurAlpha);

mix(back, front, n) is linear interpolation. When n = 0 (most pixels), the output is back — the dark background color. When n = 1 (the bright grain peaks), the output is front — a warm off-white. Values in between produce subtle grays.

The alpha channel is set to blurAlpha — the shape mask. This is the two-layer trick: inside the black hole shape, the grain plane is opaque and the particles are visible. Outside the shape, alpha drops to zero and the grain plane becomes invisible, revealing the solid dark background plane behind it.

The result: particles of light that exist only within the geometry of a black hole, flickering and drifting as the noise evolves over time. No images. No pre-rendered frames. Just math, running live on your GPU.
